Streaming Advantage

Why SSM architecture fundamentally outperforms CNN in real-time voice AI

SSM

NC-SSM Streaming

Stateful Sequential Processing
Mic → VAD → Onset Detection → Progressive Classify
  +300ms → 1st attempt (short words)
  +500ms → 2nd attempt (medium)
  +750ms → 3rd attempt (confirm)
confidence > threshold → DETECTED!
⚡
Stateful hidden state — ht accumulates temporal context naturally. No recomputation needed per window.
🎯
Progressive classification — Short words ("go") detected at 300ms, long words ("right") at 500ms. Average response: ~400ms.
🔋
Zero feature map memory — Only 720 bytes hidden state. Fits on $1.50 FPGA.
💡
Energy-aware — Silence = skip inference entirely. 0 MACs during idle. 10x battery life.

CNN Streaming (DS-CNN-S)

Stateless Window Processing
Mic → Buffer 1.0s → Full Inference → Result

Every 1s: re-buffer → re-compute entire window
No state carried between windows
Short words may be split across windows
🔄
Stateless — Each window is independent. Must recompute 24.3M MACs every time, even for same audio.
⏳
Fixed 1s buffer — Must wait full second. "go" (200ms) wastes 800ms of buffer. Minimum latency = 1000ms.
💾
196 KB feature maps — Conv2d stores all intermediate tensors. Cannot fit on low-cost FPGA.
🔋
Always computing — No speech awareness. Burns 24.3M MACs on silence. Battery drain constant.
